    CoreWeb.Cldr.Currency.pluralize

    You're seeing just the function pluralize, go back to CoreWeb.Cldr.Currency module for more information.
    Link to this function

    pluralize(number, currency, options \\ [])

    View Source

    Specs

    pluralize(pos_integer(), atom(), Keyword.t()) ::
  {:ok, String.t()} | {:error, {module(), String.t()}}

    Returns the appropriate currency display name for the currency, based on the plural rules in effect for the locale.

    Arguments

    • number is an integer, float or Decimal

    • currency is any currency returned by Cldr.Currency.known_currencies/0

    • options is a keyword list of options

    Options

    • locale is any valid locale name returned by MyApp.Cldr.known_locale_names/0 or a Cldr.LanguageTag struct returned by MyApp.Cldr.Locale.new!/1. The default is CoreWeb.Cldr.get_locale/0

    Returns

    • {:ok, plural_string} or

    • {:error, {exception, message}}

    Examples 

    iex> CoreWeb.Cldr.Currency.pluralize 1, :USD
{:ok, "US dollar"}

iex> CoreWeb.Cldr.Currency.pluralize 3, :USD
{:ok, "US dollars"}

iex> CoreWeb.Cldr.Currency.pluralize 12, :USD, locale: "zh"
{:ok, "美元"}

iex> CoreWeb.Cldr.Currency.pluralize 12, :USD, locale: "fr"
{:ok, "dollars des États-Unis"}

iex> CoreWeb.Cldr.Currency.pluralize 1, :USD, locale: "fr"
{:ok, "dollar des États-Unis"}
